検証方法論

BURNING ZEROは統計的に検証されたエッジのみを運用します

Walk-Forward OOS Out-of-Sample Validation

  • 6ヶ月学習 → 2ヶ月テスト → ローリング
  • 全結果はOut-of-Sample（学習データと評価データを完全分離）
  • 過学習を排除し、未知の市場環境での性能を評価

リーク防止 Anti-Leak Protocol

  • 7つのリークテスト全パス
  • shift(1)による未来データ遮断 — 全特徴量は1期前のデータのみ使用
  • Purged K-Fold + 48時間エンバーゴ — 学習/テスト間の情報漏洩を物理的に遮断

レジーム検証 Regime Testing

  • Bull / Range / Downtrend 各期間で個別にOOS検証
  • レジーム依存の戦略は採用しない
  • AVWAP regime artifact検証の教訓を反映 — 特定環境でのみ機能する戦略を排除

Monte Carlo 1,000回シミュレーション

  • 月次リターンのランダムシャッフルで1,000パス生成
  • 破産確率の算出 — 最悪ケースでの資金枯渇リスクを定量化
  • 5th / 25th / 50th / 75th / 95thパーセンタイル分布で期待リターン帯を把握

コスト前提 Cost Model

  • スプレッド + スリッページ込みでバックテスト実行
  • 固定コスト: 0.35 (v3 H1) / 1.5 pips (Scalp)
  • BT結果は50%割引で見積もる運用方針 — 実運用との乖離を織り込み済み
